# S03 – eda_cli: мини‑EDA для CSV

Небольшое CLI‑приложение для базового анализа CSV‑файлов. Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

* Python 3.11+
* [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта выполняется:

```bash
uv sync
```

Эта команда:

* создаёт виртуальное окружение `.venv`;
* устанавливает зависимости из `pyproject.toml`;
* устанавливает сам проект `eda-cli` в editable‑режиме.

---

## Доступные команды CLI

Приложение предоставляет две основные команды:

### `overview`

Краткий текстовый обзор CSV:

* число строк и столбцов;
* типы данных;
* пропуски;
* уникальные значения;
* примеры значений.

Пример:

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

* `--sep` — разделитель CSV (по умолчанию `,`)
* `--encoding` — кодировка файла (по умолчанию `utf-8`)

---

### `report`

Генерация полного EDA‑отчёта в указанную директорию.
Создаются:

* `report.md` — полноценный Markdown‑отчёт;
* `summary.csv` — статистика по колонкам;
* `missing.csv` — пропуски по колонкам;
* `correlation.csv` — матрица корреляций;
* `top_categories/*.csv` — top‑k категорий по категориальным признакам;
* изображения: гистограммы, тепловая карта корреляций, карта пропусков.

Базовый вызов:

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

---

## Новые параметры команды `report`

Команда поддерживает дополнительные параметры, влияющие на итоговый отчёт:

### `--top-k-categories <int>`

Сколько самых популярных значений показывать в таблицах top‑категорий.
По умолчанию: **5**.

### `--title <str>`

Заголовок, который будет записан в `report.md`.
По умолчанию: **"report"**.

---

## Пример команды с новыми параметрами

```bash
uv run eda-cli report data/example.csv \
    --out-dir reports_example \
    --sep ";" \
    --encoding "utf-8" \
    --max-hist-columns 10 \
    --top-k-categories 8 \
    --title "Полный анализ Example.csv"
```

В результате:

* будет создана папка `reports_example/`;
* в ней появятся markdown‑отчёт, csv‑файлы и изображения;
* отчёт будет иметь заголовок **«Полный анализ Example.csv»**.
